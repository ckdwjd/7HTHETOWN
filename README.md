<div>
  <h2>🛠  담당 역할</h4>
  <ul>
    <li>프로젝트 주제 선정, 기능 기획 및 DB설계</li>
    <li>전반적인 UI 및 기능구현 (메인 페이지, 마이 페이지, 관리자페이지, 로그인, 회원가입)</li>

<div>
  <h2>💡 참여 소감</h2> 
  프로젝트를 진행하면서 다양한 영역을 다뤄보았습니다. 이전에는 UI만 다루던 경험이었지만, 이번에는 DB와 서버 연동 등 다양한 부분을 다루며 많은 도전과 발견을 하게 되었습니다. UI 작업에서는 각자 다른 코드 스타일과 네이밍 방식으로 작업했던 것이 수정을 어렵게 만들었습니다. 이러한 문제점을 경험한 후에는 팀 내에서 일관된 규칙을 정해 작업 효율을 높여야겠다는 생각이 들었습니다. DB설계 역시 테이블과 컬럼을 더 구체적으로 나누는 것이 중요하다는 것을 깨달았습니다. 더 세부적인 설계와 가상 데이터 활용으로 작업 효율을 높일 수 있을 것으로 기대합니다.
  여러 코드를 합치는 과정에서 발생한 에러들을 경험하면서, 이번 프로젝트에서 Git을 활용했다면 에러를 줄일 수 있었을 것이라는 생각이 들었습니다. 다음 프로젝트에서는 Git을 적극적으로 활용하여 협업 효율을 높이려고 합니다. 프로젝트를 통해 기획과 조율의 중요성을 깨닫게 되었습니다. 각 영역 간의 조율이 프로젝트의 성패를 좌우하는 것을 몸소 체감하였습니다. 이번 프로젝트를 통해 더 나은 개발자로 성장하고자 하는 의지가 더욱 강해졌습니다.
</div>

<h2>구현 내용</h2>
<div>
  <h4>1. 메인 페이지</h4>
  <img src="7HTHETOWN Git Img/Main.png" width="600"/>
  <p>
    <h4>구현기능설명</h4>
    메인 페이지 부분 사용자가 스크를을 내렸을 때 해당 컨텐츠와 만나게 되면 애니메이션 효과를 구현
    (단, 이미 효과가 구현된 컨텐츠는 작동하지 않는다.)
  </p>
  <br>
  <h4>2. 회원가입</h4>
  <img src="7HTHETOWN Git Img/Untitled.png" width="600"/>
  <p>
    <h4>구현기능설명</h4>
    <ul> 
      <li>아이디 중복체크<br>
          아이디 인풋에 사용자가 입력 후 중복 체크 버튼을 누르게 된다면, 비동기 요청을 통해 DB에 존재하는 아이디 값을 조회 후 존재하면 ‘존재하는 아이디입니다.’, 없다면 ‘사용 가능한 아이디입니다. 사용하시겠습니까?’ 출력</li>
      <li>비밀번호 암호화<br>
          사용자가 입력한 비밀번호와 비밀번호 확인을 필터를 거쳐 암호화 작업(SHA-512)을 거친 뒤에 DB에 암호화 처리가 된 비밀번호 저장</li>
      <li>다음 주소 API 주소 검색 API를 활용하여 사용자가 주소를 직접 입력하지 않고 사용하기 편하게 구현</li>
      <li>하단 약관 동의를 체크하기 전 까지 회원가입 버튼 비활성화, 모달에 존재하는 체크박스를 체크하고 확인을 누르면 회원가입 버튼 활성화</li>
      <li>정규식을 활용하여 정합성 체크 후 알림 문구 출력</li>
    </ul>
  </p>
  <br>
  <h4>3. 로그인</h4>
  <img src="7HTHETOWN Git Img/Untitled (1).png" width="600"/>
  <p>
    <h4>구현기능설명</h4>
    사용자가 입력한 아이디 또는 비밀번호가 등록된 데이터와 일치 여부에 따라 알림 문구 구현
  </p>
  <br>
  <h4>4. 네비바(마이페이지)</h4>
  <img src="7HTHETOWN Git Img/Untitled (2).png" width="600"/>
  <p>
    <h4>구현기능설명</h4>
    로그인 성공 후 세션에 담긴 로그인 아이디가 일반 사용자인지 관리자인지 구분 후 서로 다른 메뉴바를 호출
  </p>
  <br>
  <h4>5. 마이페이지(메인)</h4>
  <img src="7HTHETOWN Git Img/Untitled (3).png" width="600"/>
  <p>
    <h4>구현기능설명</h4>
    <ul>
      <li>로그인한 회원정보를 세션에 담아와 각 항목에 해당하는 데이터 값을 넣음(회원 정보, 포인트, 배송현황)</li>
      <li>사이드바 Top값과 윈도우의 Top값을 통해 스크롤 위치가 변동될 때마다 추적하도록 구현</li>
    </ul>
  </p>
  <br>
  <h4>6. 개인 정보 수정</h4>
  <img src="7HTHETOWN Git Img/Untitled (4).png" width="600"/>
  <p>
    <h4>구현기능설명</h4>
    <ul>
      <li>수정 가능한 정보는 사용자의 이름, 이메일, 전화번호, 주소</li>
      <li>회원가입, 로그인과 마찬가지로 정규표현식을 통해 정해진 값만 들어가게 설정</li>
      <li>주소 변경은 다음 주소 API를 통해 변경이 가능하도록 구현함</li>
    </ul>
  </p>
  <br>
  <h4>7. 비밀번호 변경</h4>
  <img src="7HTHETOWN Git Img/Untitled (5).png" width="600"/>
  <p>
    <h4>구현기능설명</h4>
    변경할 비밀번호와 비밀번호 확인의 일치여부에 따라 다른 문구 출력
  </p>
  <br>
  <h4>8. 회원탈퇴</h4>
  <img src="7HTHETOWN Git Img/Untitled (6).png" width="600"/>
  <p>
    <h4>구현기능설명</h4>
    <ul>
      <li>DB에 전달한 값을 UPDATE할 수 없다면 탈퇴 실패문구 출력</li>
      <li>입력한 아이디 값이 세션에 담긴 로그인 유저의 아이디, 비밀번호와 비밀번호가 일치한다면 탈퇴 성공 문구 출력</li>
    </ul>
  </p>
  <br>
  <h4>9. 관리자 페이지(메인)</h4>
  <img src="7HTHETOWN Git Img/Untitled (7).png" width="600"/>
  <p>
    <h4>구현기능설명</h4>
    <ul>
      <li>현재 전달받은 문의 및 신고 내역의 갯수를 카운트 하여 해당 란에 표시</li>
      <li>boardLimit값을 5개로 제한하여 최근 문의 및 신고 내역만 출력</li>
      <li>사이드바 Top값과 윈도우의 Top값을 통해 스크롤 위치가 변동될 때마다 추적하도록 구현</li>
    </ul>
  </p>
  <br>
  <h4>10. 회원정보 관리 페이지</h4>
  <img src="7HTHETOWN Git Img/Untitled (8).png" width="600"/>
  <p>
    <h4>구현기능설명</h4>
    <ul>
      <li>현재 등록된 회원 리스트 출력</li>
      <li>해당 회원 정보를 클릭 시 모달창을 띄워 상세 정보 조회</li>
      <li>체크 박스를 통해 해당 회원 삭제 가능</li>
      <li>전달 받은 카테고리의 벨류 값과 검색한 키워드로 세분화된 SQL문 실행 가능</li>
    </ul>
  </p>
  <br>
  <h4>11. 고객센터 관리 페이지</h4>
  <img src="7HTHETOWN Git Img/Untitled (9).png" width="600"/>
  <p>
    <h4>구현기능설명</h4>
    <ul>
      <li>사용자가 작성한 문의 리스트 출력</li>
      <li>문의 내역을 클릭 시 모달창을 띄워 상세 정보가 나타나고 문의에 대한 답글을 달아 해당 사용자에게 전달</li>
      <li>체크 박스를 통해 해당 회원 삭제 가능</li>
      <li>전달 받은 카테고리의 벨류 값과 검색한 키워드로 세분화된 SQL문 실행 가능</li>
    </ul>
  </p>
